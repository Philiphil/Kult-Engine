Introduction

Kult-Engine is a full-stack framework
Its goal is to be a foundation design to fit every website.
Lightweight yet powerful; Kult-engine provides sets of tools for a website's common task.

Why would someone choose Kult-Engine over CodeIgniter/Laravel/Symphony 2/Slim.php... ?
 Dont.
I've made Kult-Engine for myself, because i needed a lightweight framework with only the necessary ordinary tools for a website, I didnt want tons of helpers that I wont use anyway, I didnt want over-complicated architecture that I need to learn perfectly in order to be able to hello world, I didnt want to install 3 software in order to not even make the whole thing runs but just install the stuff

Since this list goes on and on, since this list apply everywhere and since nobody seems to be bothered by those things, you probably shouldnt use Kult Engine.

Architecture

Files

Kult-Engine architecture is based upon MVC paradigm.

In the M(odel) :
  Nothing, if you want data, you gotta do them yourself.

in the C(ontrol) :
 The 4 main folders.
   ./controller/core/
    Where almost everything relevant is.
   ./controller/impt/
    Containing what you will edit
   ./controller/mods/
    Containing your mods
   ./controller/vendor/
    Containing ... well, libs.

in the V(iew) :
 3 folders and one file.
   ./www/assets/
   Where css goes
   ./www/script/
    where js goes
   ./www/images/
    where picture goes.
   ./www/invoker.class.php
   We'll talk about this in a sec.

Functioning

Main functionalitys are handle by Singleton classes

./www/invoker.class.php
 Told you we was about to talk about this file.
 Invoker is called by every page and allows the call and init of everyting needed.

./controller/impt/invokee.class.php
 Depending on the project, you might need extra call, extra init, extra const ...
 In case of such needs, Invokee is the file you have to edit.

./config.php
 config.php should be in the same folder as view/controller and model.
 Config is the ... conf file

How basic_page_using_kult_engine_works.php

<php
 require_once("invoker.class.php");
 #Invoker is require.  kult-engine\invoker::require_basics();
 #This way the whole framework is init.
 kult-engine\page::standardpage_head('Title');
 kult-engine\page::standardpage_header();
 page::standardpage_body_begin();
?>

<p> Now html</p>
<?php
 page::standardpage_body_end();
